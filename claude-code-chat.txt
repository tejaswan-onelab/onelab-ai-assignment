  ## What You're Building                                                   
                                                                            
  A **Payments Reconciliation Dashboard** — it ingests two datasets

  ---                                                                       
                                                                            
  ## Architecture                                                           
                                                                            
  ```                                                                       
  React Frontend  ←→  FastAPI Backend  ←→  In-memory / generated data       
  ```                                                                       
                                                                            
  No database needed — keep it simple. Generate data on the backend, run    
  reconciliation logic there, expose results via REST.                      
                                                                            
  ---                                                                       
                                                                            
  ## Data Model                                                             
                                                                            
  **Transactions** (what the payments platform recorded):                   
  ```                                                                       
  transaction_id, customer_id, amount, currency, timestamp, status          
  ```                                                                       
                                                                            
  **Settlements** (what the bank reported):                                 
  ```                                                                       
  settlement_id, transaction_id, settled_amount, settlement_date, batch_id  
  ```                                                                       
                                                                            
  ---                                                                       
                                                                            
  ## The 4 Planted Gaps (your job is to detect all of these)                
                                                                            
  | Gap Type | How to Plant It |                                            
  |---|---|                                                                 
  | Next-month settlement | Transaction on Jan 31, settlement date Feb 1 |  
  | Rounding difference | Transaction $10.005 → rounds to $10.00 vs $10.01  
  in two systems |                                                          
  | Duplicate entry | Same transaction_id appears twice in settlements |    
  | Orphan refund | Refund with a `ref_` prefixed ID, no matching original  
  |                                                                         
                                                                            
  ---                                                                       
                                                                            
  ## Backend (FastAPI)                                                      
                                                                            
  Three endpoints:                                                          
                                                                            
  - `GET /api/data/generate` — generates synthetic transactions +           
  settlements with all 4 gap types planted                                  
  - `GET /api/reconcile` — runs the matching logic, returns categorized     
  results                                                                   
  - `GET /api/summary` — aggregate stats (total matched, total gap value,   
  gap count by type)                                                        
                                                                            
  **Reconciliation logic:**                                                 
  1. Match transaction_id across both datasets                              
  2. Flag unmatched transactions (no settlement found)                      
  3. Flag amount mismatches (matched but amounts differ)                    
  4. Flag duplicates (settlement_id appears more than once for same         
  transaction)                                                              
  5. Flag cross-month settlements (transaction month ≠ settlement month)    
  6. Flag orphan refunds (no parent transaction found)                      
                                                                            
  ---                                                                       
                                                                            
  ## Frontend (React)                                                       
                                                                            
  Four views / sections:                                                    
                                                                            
  1. **Summary bar** — total transactions, total settled, total gaps, gap   
  value in $                                                                
  2. **Gap table** — filterable by gap type, shows transaction + settlement 
   side by side with diff highlighted                                       
  3. **Timeline view** — simple chart showing transaction date vs           
  settlement date (catches the month-end slip)                              
  4. **Raw data tabs** — show the generated transactions and settlements so 
   the assessor can verify
